# Security Policy

The Kawalia team takes security vulnerabilities seriously. We appreciate your efforts to responsibly disclose your findings, and we will make every effort to acknowledge your contributions.

## Reporting a Vulnerability

If you believe you have found a security vulnerability in Kawalia, please report it to us by emailing **fireworkrocket@126.com**.

**Please do not report security vulnerabilities through public GitHub issues.**

When reporting a vulnerability, please ensure you are using the latest version of the project. Your report should include the following information:

*   **A clear and descriptive title.**
*   **A detailed description of the vulnerability.**
*   **Steps to reproduce the vulnerability.** This could include a code snippet, a set of commands, or a description of the actions to take.
*   **Any other relevant information** that could help us understand the issue.

The maintainer will do their best to review and respond to your report in a timely manner.

## Supported Versions

Only the latest version of Kawalia is actively supported. Vulnerabilities should be reported against the most recent release.

| Version | Supported          |
| ------- | ------------------ |
| Latest  | :white_check_mark: |
| Older   | :x:                |

We appreciate your help in keeping Kawalia secure.
